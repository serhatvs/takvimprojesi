# AGENTS.md

1. Kod değişikliğinden önce `docs/agent/CURRENT_STATE.md` ve ilgili `REPO_MAP.md` bölümünü oku.
2. Tüm repository'yi gereksiz yere tekrar tarama.
3. Görev için yalnızca ilgili modül, contract ve test dosyalarını aç.
4. Yeni özellikleri dikey dilimler halinde geliştir.
5. Controller içinde iş mantığı tutma.
6. Yetkilendirmeyi yalnızca arayüzde uygulama; API tarafında zorunlu kıl.
7. Prisma şemasındaki değişikliklerde migration ve veri etkisini kontrol et.
8. Tamamlandı demeden önce lint, typecheck ve ilgili testleri çalıştır.
9. Büyük çalışmalardan sonra `CURRENT_STATE.md` dosyasını güncelle.
10. Kullanıcı istemeden commit veya push yapma.
